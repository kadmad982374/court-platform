## 1) ما يكفي بدون أي تغيير backend

النظام الحالي (Backend Phase 7 + Frontend Phase 11) **يكفي تقنيًا** لـ
pilot محدود إذا تمت الاستعدادات في §2 + §3 + §4.

السيناريوهات المغطاة:
- قيد دعوى، تعديل بيانات أساسية، استعراض، scope صحيح لكل دور.
- ترحيل جلسات، فصل، استئناف، تنفيذ، خطوات تنفيذ.
- سجل الفصل الشهري + الفلاتر + التصدير.
- مرفقات (50MB) رفع/قائمة/تنزيل.
- تذكيرات شخصية.
- إشعارات داخلية للقيد + الإسناد.
- المعرفة (مكتبة، جهات، تعاميم) قراءة + تفاصيل.
- forgot/reset password (مع OTP من logs).

## 6) معايير «نجاح الـ Pilot»

اعتبر الـ pilot ناجحًا إذا تحقق ≥ 80% من:
- [ ] لا حالة فقدان بيانات.
- [ ] لا breach scope (مستخدم يرى ما لا يجب رؤيته).
- [ ] DBA تدخل ≤ مرة/أسبوعين لتعديل user (الإسناد لم يعد يحتاج DBA).
- [ ] رضا المستخدم ≥ 4/5 على المهام اليومية.
- [ ] لا تعطّل > ساعتين متراكمة في 4 أسابيع.
- [ ] backup يعمل ويتم اختبار restore مرة على الأقل.
- [ ] متوسط زمن استجابة `< 500ms` على endpoints العمل.

عدم تحقق هذه ⇒ تأجيل التوسعة وتنفيذ mini-phase B + تحسينات §3.
