# FUNCTIONAL_SCOPE
## النطاق الوظيفي للنظام

> هذه الوثيقة تترجم §5 و§6–§18 من الوثيقة الوظيفية إلى وحدات وظيفية واضحة.
> أي ميزة لا تُذكر هنا = خارج النطاق الحالي.

---

## 1) الوحدات الوظيفية الأساسية

### 1.1 الهوية والدخول
**ماذا تفعل:** تسجيل الدخول/الخروج، إنشاء حساب، نسيان كلمة المرور وإعادة تعيينها عبر OTP، قراءة بيانات المستخدم الحالي وأدواره وعضوياته ومحاكمه المسموح بها.
**ماذا لا تفعل:** SSO خارجي، تكامل مع Active Directory، 2FA إلزامي شامل (تُعتمد OTP لاسترداد كلمة المرور فقط).

### 1.2 الهيكلية التنظيمية
**ماذا تفعل:** إدارة الفروع الـ14، الأقسام الأربعة في كل فرع، المحاكم المرتبطة بكل فرع/قسم، الغرف.
**ماذا لا تفعل:** إنشاء فروع/أقسام جديدة عبر الواجهة العامة (تُعتبر بيانات بذرية ثابتة في الإصدار الأول).

### 1.3 ضبط الوصول والصلاحيات
**ماذا تفعل:** تعيين الأدوار، تسجيل عضوية المستخدم في قسم/فرع، منح/حجب صلاحية الدخول إلى محاكم محددة، تفويض صلاحيات الموظف الإداري، تطبيق قواعد ملكية الدعوى وحدود الفرع/القسم.
**ماذا لا تفعل:** نظام صلاحيات ديناميكي حر (Permissions تبقى محصورة في القائمة المعتمدة).

### 1.4 قيد الدعوى الجديدة
**ماذا تفعل:** إنشاء دعوى مع البيانات الأساسية (§6.2 من الوظيفية)، تعديل البيانات الأساسية ضمن النطاق، إسناد الدعوى لمحامٍ، فصل مباشر استثنائي.
**ماذا لا تفعل:** السماح للمحامي بإنشاء دعوى من الصفر، أو تعديل تاريخ القيد الأول.

### 1.5 متابعة الجلسات وترحيلها
**ماذا تفعل:** ترحيل جلسة (تاريخ قادم + سبب جديد)، إظهار آخر جلسة وسبب تأجيلها وما قبلها، تاريخ كامل للجلسات (append-only)، شاشة جلسات اليوم.
**ماذا لا تفعل:** السماح بحذف/تعديل جلسة سابقة، أو تجاوز ملكية المحامي.

### 1.6 الفصل وتسجيل القرار
**ماذا تفعل:** إدخال رقم/تاريخ/نوع القرار، المبلغ والعملة، الخلاصة؛ تحويل المرحلة إلى FINALIZED.
**ماذا لا تفعل:** إنشاء أنواع قرار خارج القائمة الحصرية، أو فصل دعوى بدون مرحلة فعّالة.

### 1.7 سجل الفصل الشهري
**ماذا تفعل:** عرض الدعاوى المفصولة شهريًا حسب تاريخ القرار، بالحقول المحددة (§8.3 من الوظيفية).
**ماذا لا تفعل:** تخزين سجل فصل كجدول مستقل قابل للتعديل المباشر.

### 1.8 الانتقال إلى الاستئناف
**ماذا تفعل:** إنشاء CaseStage جديدة من نوع APPEAL مرتبطة بالمرحلة الأم، تحويل المرحلة الأم إلى read-only، إبقاء المرفقات والجلسات السابقة مقروءة.
**ماذا لا تفعل:** السماح بتعديل المرحلة السابقة بعد الانتقال.

### 1.9 الملف التنفيذي وإجراءاته
**ماذا تفعل:** إنشاء ExecutionFile مرتبط بـ LitigationCase ومرحلة المصدر، تسجيل ExecutionAction مؤرخ مع مرفقات.
**ماذا لا تفعل:** تمثيل التنفيذ كجلسات أو ربطه بـ HearingProgressionEntry.

### 1.10 المرفقات
**ماذا تفعل:** رفع/استعراض/تنزيل صور و PDF مرتبطة بمرحلة أو ملف تنفيذي أو إجراء، مع checksum وحدود حجم/نوع.
**ماذا لا تفعل:** تحرير محتوى المرفقات داخل النظام، نظام إصدارات معقد.

### 1.11 التذكيرات
**ماذا تفعل:** إنشاء تذكير مرتبط بدعوى المحامي بتاريخ/ساعة/نص، تغيير حالته.
**ماذا لا تفعل:** تذكيرات مشتركة بين عدة مستخدمين، تكرار دوري متقدم.

### 1.12 الإشعارات
**ماذا تفعل:** إشعار داخل النظام للمستخدم المعني عند قيد دعوى جديدة (الحد الأدنى)، احترام الملكية والنطاق.
**ماذا لا تفعل:** إشعارات Push/SMS/Email في الإصدار الأول (تُهيأ البنية فقط).

### 1.13 المكتبة القانونية
**ماذا تفعل:** تصنيف وتصفح وبحث بالكلمات المفتاحية، عرض محتوى يديره قسم الدراسات.
**ماذا لا تفعل:** أتمتة استيراد المحتوى من مصادر خارجية، محرر متقدم.

### 1.14 دليل الجهات العامة
**ماذا تفعل:** تصنيف بحسب الوزارات والهياكل العليا، تصفح وبحث.
**ماذا لا تفعل:** تكامل مع مصادر حكومية رسمية مباشرة.

### 1.15 القرارات والتعاميم
**ماذا تفعل:** أرشيف قرارات وتعاميم وزارة العدل وقرارات وتعاميم الإدارة، استعراض وبحث.
**ماذا لا تفعل:** workflow اعتماد/توقيع للقرارات.

### 1.16 التقارير
**ماذا تفعل:** تقارير ملخصة (مركزي/فرع/قسم/محامي) ضمن نطاق صلاحية كل دور.
**ماذا لا تفعل:** Dashboards تفاعلية متقدمة أو BI.

### 1.17 التدقيق
**ماذا تفعل:** سجل أحداث للأحداث الحساسة كاملة كما في §12 من التقنية.
**ماذا لا تفعل:** تحليل سلوك المستخدم/SIEM.

## 2) ما لا يجب اعتباره جزءًا من هذه المرحلة من المشروع
- أي وحدة لم تُذكر أعلاه.
- أي تكامل خارجي (إيميل، SMS، Push، PKI، أنظمة محاكم).
- أي وحدة "ذكاء اصطناعي" أو تلخيص آلي.
- أي إدارة مالية/فوترة/عقود.
- أي ميزة chat بين المستخدمين.
- أي workflow engine عام.
- أي تطبيق Mobile Native.
- أي ترجمة لغوية ثانية.
- أي ميزة تعتمد على افتراضات لم تُذكر صراحة في الوثيقتين.

## 3) المخرجات المتوقعة لكل وحدة (مرجع سريع)
| الوحدة | الكيان الأساسي | API الأساسي |
|---|---|---|
| identity | User | `/api/v1/auth/*`, `/api/v1/users/me` |
| organization | Branch, Department, Court | `/api/v1/branches`, `/api/v1/courts` |
| access-control | UserRole, UserDepartmentMembership, UserCourtAccess, UserDelegatedPermission | `/api/v1/users/{id}/...` |
| litigation-registration | LitigationCase, CaseStage (initial) | `/api/v1/cases` |
| litigation-progression | HearingProgressionEntry | `/api/v1/stages/{id}/rollover-hearing` |
| decision-finalization | CaseDecision | `/api/v1/stages/{id}/finalize` |
| resolved-register | (read model) | `/api/v1/resolved-register/...` |
| stage-transition | CaseStage(APPEAL), ExecutionFile (creation) | `/api/v1/stages/{id}/promote-to-appeal` , `/promote-to-execution` |
| execution | ExecutionFile, ExecutionAction | `/api/v1/execution-files/*` |
| attachments | Attachment | `/api/v1/.../attachments` |
| reminders | Reminder | `/api/v1/cases/{id}/reminders` |
| notifications | Notification | `/api/v1/notifications` |
| legal-library | LegalCategory, LegalLibraryItem | `/api/v1/legal-library/*` |
| public-entity-directory | PublicEntityCategory, PublicEntityItem | `/api/v1/public-entities` |
| circulars | Circular | `/api/v1/circulars` |
| reporting | (read models) | `/api/v1/reports/*` |
| audit | AuditEvent | داخلي + endpoints قراءة لاحقة |
